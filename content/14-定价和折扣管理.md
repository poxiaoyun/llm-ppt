## 定价与折扣管理

这一页只看三件事：折扣如何配置、折扣如何影响计费，以及如何通过统计回看这套策略是否有效。

### 折扣优惠的配置

折扣是可配置的商业策略，支持按全局、租户和更具体的业务主体逐层收口。

- 全局折扣：默认价格策略，适用于所有用户和所有请求
- 租户折扣：按客户、团队或项目单独设置，适合商务谈判和分层运营
- 更具体规则：当多个折扣同时命中时，优先使用更具体的配置

### 折扣如何影响计费

先按 Token 和模型单价算出基础费用，再通过折扣率得到最终扣减金额。

```
基础费用 = 输入费用 + 输出费用
实际费用 = 基础费用 × 折扣率
```

折扣率越低，最终扣减越少；折扣率为 1 时，按原价结算。

例如：

- 输入 1500 个 Token，单价 0.03 Points/千Token
- 输出 2000 个 Token，单价 0.06 Points/千Token
- 若折扣率为 0.8，则最终费用在基础费用上再打 8 折

### 使用统计

统计的作用不是展示图表，而是反过来验证这套折扣策略有没有真正带来经营结果。

- 按时间统计：分钟、小时、天级消耗趋势
- 按模型统计：看不同模型的价格和消耗差异
- 按用户组统计：识别团队、项目和租户差异
- 按 API Key 统计：定位具体调用主体和成本来源
