## Router（路由）
Router（路由）是 APortal 中的核心资源，负责定义请求如何匹配和转发到不同的 AI 服务提供商。
Router 就像一个智能的交通指挥系统，根据不同的规则将 API 请求转发到合适的 Provider：

- 根据请求路径分配
- 根据模型名称分配
- 根据请求头信息分配
- 支持负载均衡和故障转移

### 核心功能
#### 灵活的路由匹配
Router 支持多种匹配规则：

1. URI 匹配
- 精确匹配：完全匹配指定路径
- 前缀匹配：匹配路径前缀
- 正则匹配：使用正则表达式匹配

2. 模型名称匹配
- 根据请求中的模型名称进行路由
- 可为不同模型配置不同的 Provider
- 例如：GPT-4 请求路由到 OpenAI，Claude 请求路由到 Anthropic

3. Header 匹配
- 根据 HTTP Header 进行路由
- 支持基于版本、客户端类型等的路由策略

4. HTTP 方法匹配
- 限制允许的 HTTP 方法（GET、POST 等）

5. 负载均衡
Router 支持在多个 Provider 之间进行负载均衡：
- 自动分配请求到不同的 Provider
- 提高系统吞吐量和可用性

6. 故障转移
当主 Provider 不可用时，Router 自动切换到备用 Provider：
- 配置备用 Provider 列表
- 定义触发条件（如特定错误码）
- 提高服务可靠性

7. 高级配置
- 超时控制：设置请求超时时间
- 重试机制：配置失败重试策略
- 认证配置：控制哪些用户组可以访问
- 模型映射：将请求的模型名称映射到 Provider 的实际模型
