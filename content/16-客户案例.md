
## 客户案例｜塞力斯

**行业代表**：制造业  
**部署方式**：私有化部署

### 案例摘要

塞力斯的核心诉求不是多接几个模型，而是在制造业内网环境里，把多个品牌模型、同品牌多云账号以及自建 GPU 私有模型统一收口到同一个企业级 LLM API 入口中，形成可控、可审计、可持续运营的治理体系。

### 核心问题

- 多个模型品牌与多个云账号并存，入口分散、配置复杂，难以统一管理。
- 除公有云模型外，还要接入自建 GPU 集群上的私有模型，接入链路更长、运维复杂度更高。
- 各业务系统各自管理 API Key、路由和访问策略，权限、审计与安全边界容易失控。

### 方案落点

- 通过私有化部署，统一纳管企业自有模型账号、同品牌多云账号和本地私有模型，形成一体化 Provider 管理能力。
- 通过 Router 对不同 Provider、模型与账号统一编排，支持负载均衡、自动回退和备用账号切换。
- 在企业内网中统一配置 API Key、用户组、IP 白名单、内容安全、审计追踪和可观测能力。

### 业务价值

- 把多入口、多账号、多模型的调用体系收敛成统一治理入口，总部对业务系统的管控能力明显提升。
- 通过统一路由与自动回退机制，降低限流或异常对生产连续性的影响。
- 让模型资源管理从各系统重复接入，升级为平台化、标准化、可持续运营的长期能力。
