## 业务空间管理

Workspace 不是一个简单分组，而是从 Endpoint 开始的独立命名空间。
它的作用，是把企业里的模型调用能力拆成多个可治理、可计量、可隔离的运营单元。

一个 Workspace 通常包含下面几类核心资源：

- 独立的 Endpoint 域名和访问入口
- 独立的 Provider 配置与模型资源池
- 独立的 Router 路由策略
- 独立的用户组、API Key 与高级治理能力

### 两类业务空间

#### 1. 默认业务空间
默认业务空间由系统在注册时自动创建，适合快速开始和标准化场景：

- 使用平台统一的 Endpoint 域名
- 直接消费平台预置的模型和路由能力
- 用户不需要感知底层 Provider 与复杂配置
- 可以通过 Points 或平台套餐快速启用模型能力

适用价值：降低接入门槛，让团队先跑通业务，再逐步进入精细化治理。

#### 2. 自定义业务空间
自定义业务空间适合正式生产和复杂业务场景：

- 拥有独立的 Endpoint 访问地址
- 可以接入自己的 Provider 和 API Key
- 可以自定义 Router 规则和模型流量策略
- 所有资源都可见、可控、可审计

适用价值：让企业把自己的模型资源、权限边界和治理策略完整沉淀到平台中。

### Workspace 的隔离价值
业务空间的意义，在于把原本混在同一个账号下的调用流量拆开管理。
每个空间都可以拥有独立边界，例如：

- 独立 Endpoint，不同空间走不同访问入口
- 独立 Provider，不同空间绑定不同上游资源
- 独立 Router，不同空间执行不同流量策略
- 独立用户组和 API Key，不同空间执行不同授权范围

这样做的结果，是环境、业务、成本和权限都可以按空间治理，而不是混在一起。

### 典型使用场景

#### 环境隔离
- 生产、测试、开发使用不同的 Workspace
- 避免测试流量污染生产链路和生产账单

#### 业务隔离
- 不同事业部、项目组或客户使用独立空间
- 便于按业务边界沉淀专属模型和策略

#### 成本管理
- 按 Workspace 统计 Token 消耗、模型成本和账单口径
- 便于做预算归因和资源优化

#### 权限控制
- 按空间配置用户组、API Key、Provider 和 Router 的访问范围
- 让权限体系与业务边界保持一致

### 一句话总结
Workspace 让企业把 AI 调用能力从“账号级使用”升级为“空间级治理”，是多环境、多业务和多租户运营的基础单元。
