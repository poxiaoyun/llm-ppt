## 计费管理

APortal 的计费体系围绕 Points 统一展开。对用户来说，Points 是“可见、可算、可充值”的通用余额；对平台来说，它是把 Token 消耗、模型单价、折扣策略和财务结算统一起来的计费中间层。

### Points 是什么

Points（积分）是平台的统一计费单位：

- 调用 LLM 服务会消耗 Points
- 充值后会获得 Points 余额
- 不同法币充值会按汇率折算为 Points

这种设计的好处是把复杂的货币、币种和价格体系，收敛成一套统一的内部口径。前台看见的是余额，后台对接的是金额和账单。

### 计费规则

平台通常按 Token 消耗进行计费，并区分输入和输出两个维度：

- 输入 Token 由用户发送给模型的内容产生
- 输出 Token 由模型返回的内容产生
- 输入单价和输出单价通常不同，且由模型定价决定

#### 计费公式

```
费用 = (输入 Token 数量 × 输入单价) + (输出 Token 数量 × 输出单价)
实际费用 = 费用 × 折扣率
```

平台内部通常先按输入和输出分别统计 Token，再根据模型单价计算基础费用，最后叠加折扣率得到实际扣减金额。

#### 费用示例

假设使用 GPT-4 模型：

- 输入 1500 个 Token，单价 0.03 Points/千Token
- 输出 2000 个 Token，单价 0.06 Points/千Token
- 无折扣，折扣率为 1.0

计算如下：

```
输入费用 = 1500 × 0.03 / 1000 = 0.045 Points
输出费用 = 2000 × 0.06 / 1000 = 0.12 Points
总费用 = 0.045 + 0.12 = 0.165 Points
```

### 余额管理

#### 查看余额

在控制台中，用户可以随时查看：

- 当前可用余额
- 最近交易记录
- 充值记录和消耗记录

#### 余额不足

当余额不足时，平台会直接拦截后续调用：

- API 调用失败
- 返回 402 错误码（Payment Required）
- 需要及时充值以恢复服务

这种机制保证了计费边界清晰，避免出现超额消耗无法追踪的情况。

#### 充值方式

##### 在线支付

支持使用人民币、美元等法币充值：

- 支持多种支付渠道
- 按实时汇率转换为 Points
- 通常可即时到账

##### 优惠券充值

支持通过优惠券码充值：

- 输入优惠券码完成充值
- 自动将额度计入账户
- 可在记录中查看兑换结果

### 计费管理的业务价值

计费管理不只是“收钱”，更重要的是把消费行为标准化：

- 用户能清楚知道自己花了多少、花在什么地方
- 管理者能按团队、项目或业务空间追踪消耗
- 财务和运营可以用统一口径做结算、对账和分析

这让平台从“能调用模型”升级为“能持续经营模型使用成本”。
